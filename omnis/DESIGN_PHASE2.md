# Omnis（万象）项目设计文档 (Phase 2)

## 1. 项目定位与核心愿景
**“借酒馆之壳，塑万象之魂”**
Omnis(万象) 是基于 SillyTavern（酒馆）的智能增强中间件。其核心目标是在 **100% 保持酒馆前端体验（UI、扩展、预设）**的前提下，将静态的文字对话游戏升级为**多智能体（Multi-Agent）驱动的动态世界沙盒**。

## 2. 核心架构：双轨驱动
为了兼顾“深度推演”与“极速响应”，Omnis 采用**大小模型协同**与**动静分离**的架构：

### 2.1 透明代理层 (The Render Engine)
- **职责**：劫持酒馆对大模型（如 Claude/GPT-4）的 API 请求。
- **机制**：在转发前，将后台计算好的“当前回合近场态势”作为不可见的系统补丁注入提示词。
- **原则**：绝不破坏酒馆原有的规则书、脚本或正则表达式，采用“附加事实”而非“覆写设定”的方式。

### 2.2 影子世界引擎 (The Simulation Engine)
- **职责**：在后台异步运行世界的时间流逝（基于 Tick）与角色决策。
- **小模型驱动**：使用廉价/本地小模型（如 Qwen-1.5B/7B）专职处理 NPC 的意图推演，极大地降低计算成本。
- **全域角色提取**：突破酒馆单卡的限制，深度扫描 `character_book`，将潜藏的配角与群演全部实例化为独立的 Agent。

## 3. 进阶特性蓝图 (Phase 2)

### 3.1 分级记忆与向量增强 (Cognitive Architecture)
- **工作记忆 (Working Memory)**：记录角色最近几回合的短效见闻（如“刚听到拔剑声”）。
- **情节记忆 (Episodic Memory)**：接入酒馆向量数据库扩展（ChromaDB/Qdrant），允许角色在重返故地或遭遇旧人时，自动召回关联历史。
- **语义记忆 (Semantic Memory)**：通过阶段性反思，提炼出不可逆的性格改变或世界观认知，固化为长期设定。

### 3.2 关系图谱与社交涟漪 (Social Dynamics)
- **动态好感度网络**：维护一张二维关系图谱（A -> B 的好感度）。在同一地点的角色发生互动时，主持人模型(DM)除了推演剧情，还会输出结构化的关系变动（如“攻击导致好感度 -20”）。
- **社交感知**：角色能够感知到附近人的关系标签（如“仇人”、“亲密”），这直接影响他们下一步的意图和互动。
- **Web 可视化**：控制台实时展示角色之间发生的关系变动。

### 3.3 主动叙事钩子 (Active Narrative Hooks)
- 打破被动式“一问一答”。当后台判定重大事件发生（如刺客破门），Omnis 将主动在提示词末尾注入**剧情打断指令**，强制引导主模型渲染突发事件。

### 3.4 Omnis Dashboard (Web控制台)
- **可视化面板**：提供轻量级 Web 页面，直观展示世界态势、活跃角色清单与实时事件流。
- **管理功能**：支持配置推演频率、小模型接口、并允许随时启停特定 NPC 的自主意识。

## 4. 当前实施路线图

### 第一步：数据底座与可视化 (进行中)
1. **全域提取器**：解析 V2/V3 角色卡内置世界书，提取隐藏角色。
2. **Dashboard 原型**：搭建独立于酒馆的前端状态页与监控接口。

### 第二步：小模型真·智能体闭环
1. 剥离随机漫步逻辑，重构 `OmnisAgent`。
2. 接入小模型 API 接口，实现基于专属日志的 `Perception -> Think -> Action` 循环。
3. 引入 `Omnis DM`（主持人模型）裁决机制。

### 第三步：深度融合与高级玩法
1. **对接到 DM 判定系统**：已实现。同一地点多角色活跃时，自动调用 `OmnisLLMClient.adjudicate_interaction`，裁决结果同步更新至各角色日志。
2. **主动打断机制**：已实现。检测到“攻击、破门”等高危判定结果时，生成 `_active_hook`，并在下回合强制作为最高优先级注入到 Prompt 顶端。
3. **对接酒馆向量化存储**：待后续结合 ChromaDB 实现情节记忆。
